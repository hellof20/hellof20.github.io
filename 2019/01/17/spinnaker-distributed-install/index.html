spinnaker分布式部署模式：

环境准备

操作系统: ubuntu 14.04 或者 centos7
软件要求： docker 1.12 和 kubernetes集群1.49
最低配置要求: 2C4G

安装halyard

  1. 下载halyard的docker 容器

  1. 安装前准备

  1. 启动halyard容器

  1. 测试halyard安装是否成功

配置spinnaker

spinnaker配置部分参考单机部署手册

部署spinnaker

  1. 获取当前spinnaker的版本信息,比如获取到的是1.2.2
    $hal version list
  2. 配置版本信息
    $hal config version edit –version 1.4.2
  3. 指定分布式部署
    hal config deploy edit –type distributed –account-name $ACCOUNT

    $ACCOUNT替换成kubernetes cloud privider中的name
  4. 开始部署
    $sudo hal deploy apply
  5. 查看部署进程
    kubectl -n spinnaker get pods -o wide
  6. 新建gate和deck service
    spin-deck-remote.yaml内容如下： 
    apiVersion: v1
 kind: Service
 metadata:
   labels:
 app: spin
 stack: deck
   name: spin-deck-remote
   namespace: spinnaker
 spec:
   ports:
   - port: 9001
 protocol: TCP
 targetPort: 9000
 nodePort: 30443
      selector:
 load-balancer-spin-deck: "true"
      type: NodePort
    spin-gate-remote.yaml内容如下 
    apiVersion: v1
 kind: Service
 metadata:
      name: spin-gate-remote
      namespace: spinnaker
 spec:
      ports:
   - port: 8085
 protocol: TCP
 targetPort: 8084
 nodePort: 30444
      selector:
 load-balancer-spin-gate: "true"
      type: NodePort
    kubectl create -f spin-deck-remote.yaml
 kubectl create -f spin-gate-remote.yaml
  7. 部署完成后，默认只能本机访问，改为从外网能直接访问

    $echo “host: 0.0.0.0” | tee \
    ~/.hal/default/service-settings/gate.yml \
    ~/.hal/default/service-settings/deck.yml
    $hal config security ui edit –override-base-url http://x.x.x.x:30443
    $hal config security api edit –override-base-url http://x.x.x.x:30444
    $sudo hal deploy apply

访问spinnaker

http://x.x.x.x:30443

Troubleshooting

  1. 容器中无法直接给halyard配置代理，需要新建Dockerfile文件，把加好代理的halyard文件替换老的halyard文件并生成新的image.
    /opt/halyard/bin/halyard文件增加代理如下 
    DEFAULT_JVM_OPTS='"-Dhttp.proxyHost=web-proxy.sgp.hp.com" "-Dhttp.proxyPort=8080" "-Dhttp.nonProxyHosts=172.17.2.*" "-Dhttps.proxyHost=web-proxy.sgp.hp.com" "-Dhttps.proxyPort=8080" "-Djava.security.egd=file:/dev/./urandom" "-Dspring.config.location=/opt/spinnaker/config/"'
    Dockerfile文件如下： 
    FROM gcr.io/spinnaker-marketplace/halyard:stable
ADD halyard /opt/halyard/bin/halyard
RUN chmod +x /opt/halyard/bin/halyard
  2. front50 pod无法正常运行，需要修改yaml文件中的image，改为0.4.0-411
  3. spinnaker 1.4.2版本可以正常使用
